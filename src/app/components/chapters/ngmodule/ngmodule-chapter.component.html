NgModules help organize an application into cohesive blocks of functionality. An NgModule is a class adorned with the @NgModule decorator function. @NgModule takes a metadata object that tells Angular how to compile and run module code. It identifies the module's own components, directives, and pipes, making some of them public so external components can use them. @NgModule may add service providers to the application dependency injectors. And there are many more options covered here.

Dynamic bootstrapping with the just-in-time (JIT) compiler

the first, dynamic option, the Angular compiler compiles the application in the browser and then launches the app

main.ts
platformBrowserDynamic().bootstrapModule(AppModule);

Static bootstrapping with the ahead-of-time (AOT) compiler

Consider the static alternative which can produce a much smaller application that launches faster, especially on mobile devices and high latency networks.

main.ts
platformBrowser().bootstrapModuleFactory(AppModule);
